## Deployment options

Code Canvas Complete is a Vite/React app backed by Supabase Edge Functions. You can deploy the frontend to any static hosting provider, and the backend to Supabase's hosted infrastructure.

## Required environment variables

Configure these variables in your deployment dashboard before going live:

| Variable | Purpose |
|---|---|
| `VITE_SUPABASE_URL` | Your Supabase project URL — e.g. `https://xyz.supabase.co` |
| `VITE_SUPABASE_ANON_KEY` | Public anon key for Supabase function calls |
| `EXECUTOR_MODE` | Default execution runtime (`wandbox`, `container`, or `hybrid`) |
| `EXECUTOR_CONTAINER_BASE_URL` | Optional: URL for the container executor service |
| `EXECUTOR_CONTAINER_API_KEY` | API key for authenticating with the container executor |
| `ARDUINO_BRIDGE_TOKEN` | Token for the Arduino OTA bridge service |
| `VITE_OTA_BRIDGE_TOKEN` | Frontend-accessible OTA bridge token |

## Build commands

```bash
npm install
npm run build
```

The output lands in `dist/`. Serve it with any static host or run locally for testing:

```bash
npm run preview -- --host 0.0.0.0 --port 3000
```

## Deploy to Vercel

1. Push the repository to GitHub, GitLab, or Bitbucket
2. Create a new project in Vercel and import the repo (Vite is auto-detected)
3. Set **Build Command** to `npm run build` and **Output Directory** to `dist`
4. Add all required environment variables in the Vercel dashboard
5. Click **Deploy**

The frontend deploys in 60–90 seconds. Supabase Edge Functions must be deployed separately:

```bash
supabase functions deploy
```

## Deploy to Replit

1. Create a new Replit from the GitHub repo (or use an existing Canvas project)
2. Set the required environment variables in Replit's **Secrets** panel
3. For development, run `npm run dev` — Vite starts on port 5000
4. For a production-like preview:

```bash
npm run build
npm run preview -- --host 0.0.0.0 --port 3000
```

5. Use Replit Deployments to host the production version

## Deploy to Koyeb

1. Create a new **Web Service** in Koyeb and connect the GitHub repository
2. Configure the service:
   - **Build Command:** `npm run build`
   - **Run Command:** `npm run preview -- --host 0.0.0.0 --port $PORT`
3. Add all required environment variables
4. Deploy — Koyeb scales the service automatically

## Setting up Supabase

The backend requires Supabase Edge Functions. Deploy them with:

```bash
# Start the local emulator for testing
supabase start

# Deploy all functions to production
supabase functions deploy
```

Functions deployed include:

| Function | Purpose |
|---|---|
| `ai-chat` | Routes AI assistant requests to OpenAI, Anthropic, Gemini |
| `execute-code` | Orchestrates code execution across runtimes |
| `arduino-bridge` | Handles OTA and Bluetooth firmware uploads |
| `generate-command` | Generates terminal commands from AI prompts |
| `template-assistant` | Manages project templates and scaffolding |

All functions in `config.toml` are pre-configured with `verify_jwt = false` for AI and execution services.

## Monitoring in production

- **Supabase logs** — view edge function invocations, errors, and latency in the Supabase dashboard
- **Analytics tab** — Code Canvas shows AI spend, execution usage, and active users per team
- **Error boundaries** — the frontend catches runtime errors and shows a recovery UI

## Rollback procedure

1. Keep a backup of your environment variable values (outside `.env`)
2. Redeploy the previous Git tag to the hosting provider
3. Roll back Supabase functions with `supabase functions deploy` at the previous commit
4. Verify health by running a test AI prompt and a code execution command

<Note>
  The `VITE_SUPABASE_URL` and `VITE_SUPABASE_ANON_KEY` must match the same Supabase project across all deployment targets. Mixing projects causes auth and data access failures.
</Note>
