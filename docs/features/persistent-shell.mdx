## Shell sessions that remember

When you open a terminal in Code Canvas, you get a persistent shell session. Run `npm install`, install a Python package, or set an environment variable — it all stays in place until you explicitly reset the session or close the project.

## How persistence works

| What's preserved | Duration |
|---|---|
| Installed npm packages | Entire session |
| Python pip packages | Entire session |
| Shell environment variables | Entire session |
| Working directory (`cd` state) | Entire session |
| Command history | Across reloads |
| Terminal output log | 48 hours |

The **Session ID** shown in the terminal footer links to the full execution log. Share this ID in a support request or include it in a workflow history entry.

## How the AI uses the shell

When you ask the assistant to run a command:

1. It sends the command to the active shell session
2. Waits for the command to complete (up to 30 seconds)
3. Captures stdout and stderr
4. Displays the output in the chat and continues the conversation with that context

The assistant can chain multiple commands in one response:

```
1. npm install missing-package
2. npm run build
3. Check for errors and suggest fixes
```

## Terminal outputs in context

Pin any terminal output card to keep it visible while chatting. The AI sees pinned outputs as part of its workspace context. Click **Summarize** to have the assistant compress a long output into key points.

## Supporting services

Additional Supabase functions run alongside the shell executor to support AI workflows:

| Function | What it does |
|---|---|
| `ai-chat` | Routes prompts to AI models and returns structured responses |
| `generate-command` | Converts natural language to safe shell commands |
| `template-assistant` | Clones and scaffolds project templates |
| `execute-code` | Routes code to the right executor (WebContainer/Wandbox/Container) |

All these services share the same session state, so your context is consistent across shell commands, AI prompts, and code execution.

## Long-running commands

The default execution timeout is **30 seconds**. For tasks that take longer:

- **Stage them in a workflow** — workflows can run for several minutes
- **Use `nohup` or background processes** — `nohup npm run build &`
- **Ask the assistant to break it into steps** — it can split a long task into resumable pieces

## Arduino and hardware uploads

When you flash a board, the hardware bridge takes over from the regular shell:

- `avrdude` handles AVR (Uno, Nano, Mega) uploads
- `esptool` and `espota.py` handle ESP (ESP32, ESP8266) uploads
- Progress is shown in a dedicated upload progress bar (not the regular terminal)
- The bridge reports binary size, transfer rate, and success/fail status

If a required uploader is missing, Code Canvas shows a tooltip with the exact install command. The assistant can run the install for you.
