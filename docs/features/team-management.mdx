## Creating a team

From the IDE header, open **Team Admin** to create a new team:

1. Click **Create Team**
2. Enter a name and description
3. Choose an icon using the format `iconName::#hexColor` (Lucide icons with custom accent color)
4. Click **Create**

You're assigned as the **Owner** automatically.

## Member roles

| Role | What they can do |
|---|---|
| **Owner** | Full control — billing, policies, members, delete team |
| **Admin** | Manage members, enforce policies, view analytics, create templates |
| **Member** | Access team projects, use shared templates, submit forms |

### Inviting members

1. Open **Team Admin → Members**
2. Click **Invite Member**
3. Enter the member's email address
4. Select their role
5. Click **Send Invite**

The invitee receives an email and sees a pending invitation badge in their dashboard. Once accepted, they appear in the member list with their assigned role.

## Policies

Policies enforce rules across all team members. Toggle any policy on or off without deleting it:

| Policy | Description |
|---|---|
| **Allowed languages** | Restrict which programming languages members can create projects in |
| **Required template** | Force all new projects to use a specific starter template |
| **AI model restrictions** | Limit which model tiers (Pro, Flash, Lite) members can access |
| **Spending limits** | Set per-member monthly spend caps in your currency |

### Spending limits

When a member hits their monthly cap:
- AI requests return a quota-exceeded error
- Server-side code execution is paused
- The admin receives a notification
- The cap resets at the start of the next billing cycle

Admins can manually override a cap for a specific member in an emergency.

## Custom templates

Team admins build shared starter templates visible to all members:

1. Open **Team Admin → Templates → New Template**
2. Fill in:
   - **Name** and description
   - **Language** (used to set up syntax highlighting and toolchain)
   - **Icon** (Lucide icon name)
   - **File tree** — add files and folders with starter code
3. Toggle **Required** if all new team projects must use this template
4. Click **Save Template**

Members see team templates in the new-project dialog alongside the global template library.

## Forms

Build custom forms for onboarding, project intake, or team surveys:

1. Open **Team Admin → Forms → New Form**
2. Add fields:
   - **Text** — single or multi-line
   - **Number** — with optional min/max
   - **Select** — dropdown with predefined options
   - **Checkbox** — boolean toggle
3. Mark fields as required or optional
4. Publish the form — team members see it in the Forms section

Form responses are collected with timestamps and user attribution. Export responses as CSV from the form admin view.

## Analytics & spending

The analytics tab gives admins visibility into team activity:

| Metric | Details |
|---|---|
| **Total spend** | Breakdown by AI model, execution type, member, and date |
| **AI usage** | Requests per model tier, token counts, average response time |
| **Execution usage** | Runtime hours, execution count by language |
| **Active users** | Daily/weekly active users, session lengths |
| **Project activity** | Commits, file edits, workflow runs per project |

## Approval workflows

Require admin sign-off before sensitive actions execute:

1. Open **Team Admin → Policies → Approvals**
2. Enable approvals for:
   - Production deployments
   - Arduino firmware flashing
   - Database migrations
   - AI commands that touch production secrets
3. Designated approvers receive a notification and click **Approve** or **Deny**

The action waits in a pending state until approved. If denied, the requester sees the reason.

## Deleting a team

Only the **Owner** can delete a team. All projects, templates, forms, and run history are permanently deleted. This action cannot be undone.

From **Team Admin → Settings → Danger Zone**, click **Delete Team** and confirm with the team name.
