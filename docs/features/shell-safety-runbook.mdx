# Terminal Safety Runbook

Use this runbook to reduce avoidable outages, data loss, and expensive mistakes when working in a shell.
It is intentionally procedural: each command profile has a purpose, common failure modes, preflight checks, and rollback ideas.

## Universal preflight checklist
- Confirm your host and account first (`hostname`, `whoami`, and active cloud profile).
- Print current path and list nearby files before destructive commands (`pwd`, `ls -la`).
- Prefer dry-run or read-only modes before write modes.
- Capture a timestamped command log during maintenance windows.
- Create restorable backups before changing stateful systems.
- Use explicit paths and avoid broad wildcards for mutation commands.
- Add command timeouts for network calls and long-running jobs.
- Verify exit codes and run post-change validation checks.

## Command profiles
Each profile is intentionally short so operators can scan quickly during incidents.

### 001. `cd`
- **Risk level:** Moderate.
- **Why it matters:** `cd` is frequently used in automation and manual operations; misuse can cause cascading operational impact.
- **Safe usage pattern:** Run `cd --help` (or manual page) first in unfamiliar environments, then execute with explicit targets and logged output.
- **Preflight checks:** verify working directory, permissions, target scope, and expected side effects before running `cd`.
- **Post-change validation:** inspect command exit status, review changed resources, and run a service-level check relevant to `cd` outcomes.
- **Rollback hint:** maintain snapshots/backups so actions involving `cd` can be reversed quickly without improvisation.
