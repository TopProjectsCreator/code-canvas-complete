## What is the API playground?

The API playground is a full-featured HTTP client built into Code Canvas. Craft requests, test APIs, inspect responses, and save templates — all without leaving the workspace. It's connected to the workflow system so every saved request can become a workflow step.

## Building a request

1. Open the **API** panel from the sidebar
2. Select a method: `GET`, `POST`, `PUT`, `PATCH`, `DELETE`
3. Enter the URL
4. Add headers, query parameters, and a request body
5. Click **Send** to execute

## Authentication options

The playground supports all common auth patterns:

| Auth type | Config |
|---|---|
| **Bearer token** | Enter a token — applied as `Authorization: Bearer <token>` |
| **Basic auth** | Enter username and password — base64-encoded automatically |
| **API key** | Add as a header or query parameter |
| **OAuth 2** | Enter client ID and secret — the playground handles the token exchange |
| **No auth** | Public endpoints |

Store sensitive tokens in your workspace **Environment** and reference them as `{{env.MY_API_KEY}}` so they're never hardcoded in saved requests.

## AI-assisted payload building

The assistant can read OpenAPI/Swagger specs and fill request bodies automatically:

- **"Send a GET request to the /users endpoint with auth"** — builds the full request with headers
- **"Create a POST to create a new order with a sample body"** — generates a realistic test payload
- **"What does this API response mean?"** — explains the response fields in plain English

Paste an OpenAPI spec URL into the assistant chat and it can pre-populate the entire request for any endpoint.

## Saving and organizing requests

After a successful request:

1. Click **Save as template**
2. Give it a name (e.g., "Get current user") and optional folder
3. Saved templates appear in the left sidebar for quick re-use

Templates support variable interpolation: `{{userId}}` becomes a prompt-able field when you run the template.

## Linking to workflows

Saved requests can be dragged directly into workflow steps:

1. Open a workflow in the workflow builder
2. Click **Add step → API Call**
3. Select a saved template from the dropdown, or configure a new request inline

The step sends the request as part of the workflow run. The response is captured as `prev.body`, `prev.status`, and `prev.headers` for the next step to use.

## Inspecting responses

After sending a request, the response pane shows:

- **Status code** with color indicator (green for 2xx, red for 4xx/5xx)
- **Response body** — formatted JSON or raw text
- **Headers** — all response headers in a collapsible list
- **Timing** — total request duration and breakdown (DNS, TLS, first byte, download)
- **Size** — response body size in bytes

## GraphQL support

Switch the request type to **GraphQL** to get:

- Schema introspection with autocomplete for fields and arguments
- Variable substitution panel
- Response viewer with inline type documentation
