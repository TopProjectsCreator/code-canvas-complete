## Publish in stages, not all at once

Staged releases reduce risk and make fixes easier.

## Recommended release plan

1. **Private preview** (you + trusted testers)
2. **Small public release** (limited audience)
3. **Full release** after quick fixes

## Pre-launch checklist

- Main user flow works
- No obvious typos/placeholders
- Mobile + desktop checked
- Performance acceptable on first load
- Description and thumbnail ready

## Rollback plan (must-have)

Before launching, decide:

- Which prior version is safe to roll back to
- Who triggers rollback
- Where status updates will be posted

## Launch-day routine

- Monitor feedback for first 30–60 minutes
- Fix critical issues quickly
- Post concise changelog update

## Post-launch growth loop

- Track where users drop off
- Prioritize most requested improvement
- Ship small updates frequently
