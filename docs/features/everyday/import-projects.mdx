## Import without terminal setup

You do not need command-line knowledge to bring an existing project into Code Canvas.

## 5-step import flow

1. Open **Import Project** from the home/start screen
2. Choose **GitHub**, **GitLab**, or **Bitbucket**
3. Approve account access
4. Select repository + branch
5. Click **Open in Workspace**

## First things to check after import

- Open the `README`
- Find dependency file (`package.json`, `requirements.txt`, etc.)
- Run the default start command
- Confirm the preview launches

## If this is your first time importing

Start with a smaller project so your first setup is quick and confidence-building.

### Beginner-friendly first edit ideas

- Update app title text
- Change one color
- Edit an “About” paragraph
- Replace a placeholder image

## Common problems and easy fixes

### “I can’t see my repo”
- Reconnect the provider account
- Confirm repository permissions
- Check if the repo is in a different organization

### “Import is slow”
- Start with default branch (`main`)
- Skip huge binary folders in first pass
- Let indexing finish before searching files

### “Project opens but won’t run”
- Install dependencies shown in README
- Confirm runtime version (Node/Python/etc.)
- Check environment variables in settings

## Safety habits for non-technical teams

- Create a new branch before making edits
- Commit small changes frequently
- Use clear commit messages (“Update welcome text”, etc.)
- Open pull requests for review instead of pushing directly to main
