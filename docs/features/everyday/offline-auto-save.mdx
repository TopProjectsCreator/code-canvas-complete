## Good news: your work can continue offline

If your connection drops, Code Canvas can still preserve ongoing edits locally. This is especially helpful for travel, unstable Wi-Fi, or temporary outages.

## What auto-save does

- Stores edits locally
- Marks unsynced changes clearly
- Re-syncs when connection returns
- Prompts to restore after accidental tab close/crash

## Recovery checklist (after reconnecting)

1. Reopen your project
2. Accept restore prompt if shown
3. Review changed files list
4. Resolve any duplicate/merge prompts
5. Commit once stable

## Practical habits that prevent data loss

- Save naturally after meaningful edits
- Keep file names clear so restores are easy to review
- Avoid clearing browser storage during active projects
- Export especially important files before long offline sessions

<Warning>
Changes made offline on one device do not magically appear on your other device until sync succeeds.
</Warning>

## How to handle sync conflicts calmly

If both local and remote versions changed:

- Compare both versions side by side
- Keep the most recent correct content
- Re-run a quick functional check
- Commit with a note like “Resolve sync conflict”

## Best workflow for travel days

- Work in focused chunks
- Commit locally after each chunk
- Keep a short TODO list for online-only tasks (push, PR, deployment)
