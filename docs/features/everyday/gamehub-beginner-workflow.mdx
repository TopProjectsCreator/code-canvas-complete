## Start simple and finish something

Most beginners quit when they start too big. The easiest path is:

1. Pick a starter template
2. Make a small visual change
3. Add one feature
4. Test it
5. Share it

## 60-minute first build plan

### 0–15 min: Setup
- Open template
- Review folder names
- Run preview once

### 15–35 min: Personalize
- Rename project
- Change colors/images/text
- Adjust one interaction

### 35–50 min: Add one enhancement
- New score rule, new button action, or extra scene

### 50–60 min: Test + share
- Quick test pass
- Capture screenshot
- Send preview to a friend

## Scope rules that keep beginners successful

- Limit first version to one core idea
- Avoid adding too many screens at once
- Prefer clarity over flashy complexity

## Simple quality checklist

- Loads reliably
- Main action works
- No obvious broken button/link
- Title and instructions are clear
