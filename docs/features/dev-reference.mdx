## System architecture

Code Canvas Complete is a Vite/React single-page app (SPA) backed by Supabase Edge Functions for AI, execution, and hardware services.

```
Browser (Vite/React)
  ├── src/pages/         ← Route pages (IDE, Docs, Landing)
  ├── src/components/    ← Reusable UI (editor, AI panel, hardware, etc.)
  ├── src/contexts/      ← Shared state (Auth, Theme, AI, MCP, Env)
  ├── src/hooks/         ← Custom React hooks
  └── src/lib/           ← Utility functions and API clients

Supabase Edge Functions
  ├── ai-chat            ← Multi-model AI chat routing
  ├── execute-code       ← Code execution orchestration
  ├── generate-command   ← NL → shell command conversion
  ├── template-assistant ← Project template management
  └── arduino-bridge     ← OTA and Bluetooth upload bridge

External services
  ├── Wandbox            ← Cloud compiler (Go, Rust, C++, etc.)
  ├── Godbolt            ← Arduino AVR/ESP compilation
  ├── OpenAI / Anthropic / Gemini ← AI model providers
  └── Meshy / Tripo / Fal.ai     ← 3D model generation
```

| Technology | Purpose |
|---|---|
| **Vite** | Build tool and dev server (port 5000) |

| **Tailwind CSS** | Utility-first styling |

Key source folders:

- `src/components/ai/` — AI chat panel, widget catalog, comparison panel
- `src/components/ide/` — Editor tabs, file tree, terminal, settings dialog
- `src/components/scratch/` — Scratch VM integration
- `src/components/arduino/` — Breadboard designer, upload UI
- `src/components/office/` — Word/Excel/PowerPoint editors
- `src/contexts/` — ThemeContext, AuthContext, MCPContext, EnvContext
- `src/pages/` — App routing (App.tsx, Index.tsx, Docs.tsx, Landing.tsx, NotFound.tsx)

## Backend services

### execute-code function

Routes code execution to the correct runtime:

```typescript
POST /functions/v1/execute-code
{
  "code": "print('hello')",
  "language": "python",
  "mode": "container" | "wandbox" | "hybrid",
  "sessionId": "optional"
}
```

Configured via:
- `EXECUTOR_MODE` — default executor
- `EXECUTOR_CONTAINER_BASE_URL` — container orchestrator endpoint
- `EXECUTOR_CONTAINER_API_KEY` — container auth key

### ai-chat function

Routes AI assistant requests to the right model provider:

```typescript
POST /functions/v1/ai-chat
{
  "messages": [...],
  "model": "gpt-4o" | "claude-3-5-sonnet" | "gemini-1.5-pro",
  "systemPrompt": "...",
  "mcpContext": [...]
}
```

### arduino-bridge function

Handles firmware upload coordination:

```typescript
POST /functions/v1/arduino-bridge
{
  "method": "ota" | "bluetooth",
  "boardIp": "192.168.1.100",
  "binary": "<base64>",
  "token": "..."
}
```

Secured by `ARDUINO_BRIDGE_TOKEN` / `VITE_OTA_BRIDGE_TOKEN`.

## CLI commands

```bash
# Development
npm install           # Install all dependencies
npm run dev           # Start Vite dev server on port 5000
npm run build         # Production bundle to dist/
npm run preview       # Preview built bundle

# Code quality
npm run lint          # Run ESLint
npm run type-check    # Run TypeScript type checking
npm run test          # Run Vitest test suite

# Supabase
supabase start        # Start local Supabase emulator
supabase functions serve  # Run functions locally
supabase functions deploy  # Deploy all functions

# Docs (from docs/ directory)
mint dev              # Preview docs site locally
mint broken-links     # Check for broken links
mint validate         # Validate docs structure
```

## Adding a new route

1. Create a page component in `src/pages/MyPage.tsx`
2. Add the route to `src/App.tsx`:
   ```tsx
   <Route path="/my-page" element={<MyPage />} />
   ```
3. Link to it from the relevant navigation component

## Adding an AI widget

1. Create a widget component in `src/components/ai/widgets/`
2. Register it in the widget catalog in `src/components/ai/WidgetCatalog.tsx`
3. The assistant will surface it in chat when relevant

## Extension points

### Workflow hooks

The workflow system is defined in `src/hooks/use-workflow.ts`. Each step type is a typed handler:

```typescript
type WorkflowStep =
  | { type: 'ai-prompt'; prompt: string }
  | { type: 'terminal'; command: string }
  | { type: 'api-call'; url: string; method: string; body: unknown }
  | { type: 'file-write'; path: string; content: string }
```

### MCP server integration

MCP server state lives in `src/contexts/MCPContext.tsx`. Add new server types or custom parsers there.

### Theme tokens

Themes are defined in `src/contexts/ThemeContext.tsx` as CSS variable maps. Add new tokens to the theme interface and update `src/styles/themes.css` to use them.
